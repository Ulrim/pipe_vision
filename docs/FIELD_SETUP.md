# AIVIS 현장 셋업 가이드 (광학 배치 · 캘리브레이션 · 다중 튜브 모드)

현장 담당자(비개발자 포함)를 위한 **따라하기 가이드**다. 시뮬레이터/합성 이미지 기반의
기능 검증은 이미 끝났다. **실물에서의 길이·표면 판정 정확도는 이 문서의 광학 셋업과
캘리브레이션을 얼마나 정확히 수행하느냐에 달려 있다.**

관련 문서:
- 라즈베리파이 설치·운영: `docs/RASPBERRY_PI.md` (워커 실행, systemd, 촬영 레시피 상세)
- 데이터 수집·라벨링 규격: `CLAUDE.md` 부록 A

> 소유: devops(docs). 이 문서는 설치·운영 절차만 다루며 애플리케이션 소스는 변경하지 않는다.

---

## 1. 개요 — 왜 광학 셋업이 전부인가

AIVIS 는 카메라 없이도 전체 파이프라인이 동작하도록 만들어져 있고(HAL + 시뮬레이터),
합성 이미지 기준 검증(분할·길이·판정·저장·화면)은 통과한 상태다. 남은 변수는 딱 하나,
**현장의 빛과 기하(거리·각도)** 다.

- 길이 측정은 "픽셀 거리 × 보정계수(px→mm)" 방식이다. **카메라 거리·초점이 흔들리면
  보정계수가 틀어져 길이가 통째로 틀린다.**
- 표면 판정(유분기/변색/스크래치)과 다중 튜브 분리는 밝기 패턴을 본다. **정반사(번쩍임),
  외광 변화, 배경의 금속 칩은 그대로 오검으로 이어진다.**

아래 순서대로 진행한다: **배치(§2) → 조명/차광(§3) → 레시피 고정(§4) → 캘리브레이션(§5)
→ (필요 시) 다중 튜브 모드(§6) → 최종 점검(§8)**.

## 2. 카메라 배치

1. **윗면 직하방(수직 하향) 설치.** 카메라가 튜브가 놓인 면을 정확히 수직으로 내려다보게
   한다. 기울어지면 원근 때문에 한쪽 끝이 길거나 짧게 측정된다.
2. **튜브 양 끝단이 모두 프레임 안에 들어와야 한다.** 길이 측정은 화면에 보이는 양 끝
   엣지 사이 거리로 계산하므로, 끝이 잘리면 측정 자체가 불가능하다. 가장 긴 품목 기준으로
   좌우 여유를 두고 시야를 잡는다.
3. **튜브 축과 프레임 축 정렬.** 튜브가 화면의 가로(또는 세로) 방향과 나란히 눕도록
   V홈/받침 치구로 자세를 고정한다. 다중 튜브 모드는 기본적으로 "가로로 누운 튜브"를
   가정한다(§6).
4. **거리 고정(브래킷 필수).** 카메라는 손이나 임시 거치가 아니라 **볼트 고정 브래킷**으로
   설치한다. 거리가 1mm 만 변해도 px→mm 스케일이 변한다. 거리를 바꾸면 반드시
   재캘리브레이션(§5).

### 2.1 시야(FOV) 대략 계산 예

필요 시야 = 최장 튜브 길이 + 좌우 여유(각 10~15%).

예) 튜브 250mm, 여유 25mm×2 → **필요 시야 약 300mm**.

Raspberry Pi Camera Module 3(표준, 수평 화각 약 66°) 기준 대략:

```
촬영거리 ≈ 필요시야 ÷ (2 × tan(33°)) ≈ 300mm ÷ 1.30 ≈ 약 230mm
```

이때 해상도 2304px 가로 기준 약 0.13mm/px 이며, 엣지 검출은 서브픽셀 보간을 쓰므로
길이 반복성은 이보다 좋다. 위 수치는 **출발점**일 뿐이며, 실제 거리는 설치 후
캘리브레이션(§5)으로 확정한다.

## 3. 조명 / 차광 — 정반사와 금속 칩이 오검의 주원인

1차 현장 사진 검토 결과, **금속 표면의 정반사(번쩍이는 하이라이트)와 베드 위 절단 칩**이
오검(스크래치/유분기 오탐, 튜브 분리 실패)의 가장 큰 원인으로 확인됐다. 다음을 지킨다.

| 항목 | 해야 할 것 | 하지 말 것 |
|---|---|---|
| 광원 | **확산광**(면광원, LED 바 + 디퓨저, 돔조명) | 맨 LED/스팟 **직사광 금지**(정반사 하이라이트 생성) |
| 외광 | **차광 후드/암실**로 검사부를 덮어 창문·천장등·용접광 차단 | 시간대·날씨에 따라 밝기가 변하는 개방 설치 |
| 배경 | **무광 어두운 배경판**(흑/짙은 회색, 무광 도장 또는 무광 시트) | 반짝이는 금속 베드 노출, 밝은 배경 |
| 청결 | 검사부 베드의 **금속 칩·분진을 매 교대 청소** | 칩이 쌓인 상태로 검사(칩이 스크래치/이물로 오검됨) |
| 고정 | 조명 위치·밝기를 브래킷으로 **고정** 후 표시(마킹) | 조명을 수시로 옮기거나 밝기 조절 |

- 스크래치 검출 강화가 필요하면 **저각도 사광(raking light)** 을 보조로 쓸 수 있으나,
  이 역시 위치·각도를 고정하고, 고정 후 반드시 캘리브레이션·판정 확인을 다시 한다.
- 조명을 바꿨다면(위치/밝기/종류) → **§4 레시피 재확인 + §5 재캘리브레이션**.

## 4. 촬영 레시피 고정 — 자동노출·자동초점 금지

촬영 파라미터는 코드에 하드코딩되지 않고 **대시보드 → 기준정보 → 품목 편집 →
"촬영 레시피 (JSON)"** 란에 저장된다(`item_master.capture_recipe`). 워커가 기동할 때
이 레시피를 읽어 카메라(PiCameraAdapter)에 그대로 적용한다.

Raspberry Pi Camera Module 3 기준 예시(값은 현장 실측으로 확정):

```json
{
  "exposure_us": 4000,
  "analogue_gain": 1.5,
  "af_mode": "manual",
  "lens_position": 4.2,
  "width": 2304,
  "height": 1296
}
```

| 키 | 의미 | 왜 고정하나 |
|---|---|---|
| `exposure_us` | 노출 시간(마이크로초) | 자동노출이면 프레임마다 밝기가 흔들려 표면 판정·엣지 검출이 불안정 |
| `analogue_gain` | 아날로그 게인(배수) | 노출과 함께 밝기를 고정. 포화(하이라이트 날림)를 피하는 선에서 최소로 |
| `af_mode` | `"manual"` 고정 | **자동초점(Continuous AF)은 절대 금지.** 프레임마다 초점거리가 미세하게 변해 px→mm 스케일이 흔들리고 길이 오차를 만든다 |
| `lens_position` | 수동 초점 위치(대략 1/거리(m)) | 촬영거리에서 가장 선명한 값을 실측으로 찾아 고정 |
| `width`/`height` | 캡처 해상도 | 해상도를 바꾸면 px→mm 스케일이 달라짐 → 반드시 재캘리브레이션 |

절차:

1. 대시보드 로그인(품질관리자 이상) → **기준정보** 화면 → 해당 품목 [편집].
2. "촬영 레시피 (JSON)" 란에 위 형식으로 입력 → 저장(버전이 자동 증가하고 이력이 남는다).
3. **워커 재시작** — 워커는 기동 시 1회 기준정보를 읽으므로, 레시피(또는 개수 등 기준정보)
   변경 후에는 워커를 재시작해야 반영된다:

   ```bash
   sudo systemctl restart aivis-vision
   ```

4. HMI 에서 화면 밝기·선명도를 확인. 하이라이트가 하얗게 날리면 `exposure_us` 또는
   `analogue_gain` 을 낮추고, 흐리면 `lens_position` 을 미세 조정한다.

> "고정 3종 세트": **수동 초점(lens_position) + 고정 촬영거리(브래킷) + 고정 노출**.
> 셋 중 하나라도 바꾸면 §5 캘리브레이션을 다시 한다.

## 5. 웹 캘리브레이션 절차 (px→mm 보정)

준비물: **실제 길이를 정확히 아는 기준 파이프 1개**(버니어캘리퍼스 등으로 실측해 둔 것)
또는 게이지. 공구가 없으면 품질담당이 실측한 값을 사용한다.

### 따라하기

1. **기준 파이프 1개를 검사 위치에 놓고 검사를 1회 돌린다.**
   (평소처럼 트리거가 걸리면 워커가 자동으로 촬영·측정한다.)
2. **HMI 또는 대시보드(검사이력)에서 시스템 측정값(mm)을 확인**해 적어둔다.
   예: 시스템 측정값 249.2mm, 실제값(실측) 250.0mm.
3. **대시보드 → 기준정보 → 해당 품목 [편집] → 아래쪽 "캘리브레이션" 섹션**에서:
   - `측정값(measured_mm)` 에 시스템이 측정한 값 입력 (예: 249.2)
   - `실제값(actual_mm)` 에 실측 길이 입력 (예: 250.0)
   - 화면에 "예상 새 계수" 미리보기가 뜬다 → **[보정]** 버튼 클릭.
   - 시스템이 `px→mm 계수 = 기존 계수 × (실제값 ÷ 측정값)` 으로 자동 보정하고,
     변경 이력(버전·작업자·전후 값)을 기록한다. 권한: 품질관리자 이상.
4. **같은 파이프를 다시 검사**해 측정값이 실제값에 들어왔는지 확인한다.
   오차가 공차 대비 유의하면(예: 공차의 1/10 초과) 3번을 한 번 더 반복한다.

### 반복성 확인 (필수)

같은 기준 파이프를 **10회 반복 검사**하고 대시보드 검사이력에서 10개 측정값을 본다.

- 최대값 − 최소값(산포)이 **허용 공차의 약 1/10 이내**면 양호.
- 산포가 크면 → 초점/거리/조명이 흔들리는 것이다. §2~§4 를 재점검한다
  (특히 자동초점이 꺼져 있는지, 브래킷이 단단한지).

> 재캘리브레이션이 필요한 경우: 카메라/조명을 옮겼을 때, 해상도(`width/height`)나
> `lens_position` 을 바꿨을 때, 치구를 교체했을 때, 품목 신규 등록 시.

## 6. 다중 튜브 모드 운영 (한 프레임에 N개)

한 번의 촬영으로 나란히 눕힌 튜브 여러 개를 동시에 검사하는 모드다. 별도 프로그램 교체
없이 **기준정보의 개수 값 하나로 전환**된다.

### 켜는 법

1. 대시보드 → 기준정보 → 품목 [편집] → **"개수(오더당 튜브)"** 에 N 입력
   (1 = 단일 모드, 2 이상 = 다중 모드, 최대 20). 필요 시 **외경(mm)** 도 함께 입력.
2. 저장 → **워커 재시작**(§4와 동일, 워커는 기동 시 기준정보를 읽는다).
3. 이후 워커는 자동으로 **배치 모드**로 동작한다: 1프레임 촬영 → 튜브 N개로 분리 →
   각 튜브를 개별 검사 → 튜브별 결과 N건을 저장(원본/결과 이미지는 배치당 1장 공유,
   튜브 번호로 구분).

### 화면에서 보는 것

- **결과 이미지**: 튜브마다 색 박스(OK 초록/NG 빨강)와 번호(#1, #2, …)가 그려지고,
  상단 바에 `BATCH: 판정  tubes 검출수/기대수  NG 수` 요약이 표시된다.
- **HMI**: 같은 배치의 튜브들이 **배치 카드 1장**으로 묶여 나온다 — 대표 이미지 +
  튜브별 그리드(#번호, OK/NG, 불량코드, 길이mm). NG 튜브는 빨강 강조, 눌러서 재확인 입력.

### 개수 불일치 경고의 의미

시스템은 기대 개수와 **별도로 독립 자동 검출**을 수행해 개수를 센다. 검출 개수가 기준정보의
기대 개수와 다르면 **개수 불일치**로 표시되고, 배치 전체 판정이 NG 가 된다. 원인은 둘 중 하나:

1. **실제로 개수가 다르다** (투입 누락/과다) → 현물 확인.
2. **분리(seam) 검출이 틀렸다** (조명/배경 문제) → §7 로 이동.

### 배치 요건 (반드시 지킬 것)

- 튜브가 **서로 붙어 있어도 된다**(밀착 허용). 단:
  - **단일 층**으로만 놓는다(겹쳐 쌓기 금지).
  - 모든 튜브가 **서로 평행**하게, 프레임 축과 나란히.
  - **모든 튜브의 양 끝이 프레임 안**에 들어와야 한다(길이 측정 전제).
- 배경은 §3의 무광 어두운 배경판. 튜브(밝음)와 배경(어두움)의 대비가 분리의 핵심이다.

## 7. seam(튜브 경계) 검출 현장 보정

다중 모드의 분리는 "밝은 마루(crown) — 어두운 골(seam)" 밝기 패턴을 이용한다. 실물에서
분리가 틀리면 **먼저 §3 조명/배경을 의심**하고, 그래도 안 되면 아래 파라미터를 공급기업과
협의해 조정한다. (파일: `services/vision/multi/segment.py` — 코드 수정은 공급기업 담당)

| 파라미터(코드 내 상수) | 현재값 | 의미 |
|---|---|---|
| 전역 대비 컷 | `gray.max() − gray.min() < 20` → 튜브 없음 처리 | 프레임 전체 명암 차가 20 계조 미만이면 빈 프레임으로 간주 |
| seam 돌출도 문턱 | `prom_frac = 0.10` (프로파일 범위의 10%) | 자동 모드에서 이보다 얕은 골은 seam 으로 안 봄 |
| 프로파일 계산 구간 | 길이축 **중앙 60%** (양끝 20%씩 제외) | 짧은 튜브의 배경 혼입 방지 |
| 전경 커버리지 문턱 | `0.3` (행/열의 30% 이상이 전경) | 튜브 밴드·길이 범위 판정 기준 |
| seam 최소 간격 | 자기상관 피치의 **0.5배** | 한 골에서 seam 이 중복 선택되는 것 방지 |
| 기대개수 스냅 창 | 기대 피치의 **±0.4배** | 기대 위치 근방에서만 골을 탐색 |
| 최대 튜브 수 | `MAX_TUBES_HARD = 20` | 하드 상한 |

### 증상별 가이드

| 증상 | 흔한 원인 | 현장에서 먼저 할 것 | 조정 지점(공급기업) |
|---|---|---|---|
| **과분할** (실제보다 많이 검출, 튜브가 반쪽으로 쪼개짐) | 정반사 하이라이트가 튜브 표면에 가짜 "골"을 만듦 | 직사광 제거, 확산광/차광 강화(§3) | `prom_frac` 상향(예 0.10→0.15) |
| **미분할** (실제보다 적게 검출, 두 개가 하나로 붙음) | 밀착 튜브 사이 골이 너무 얕음(과노출로 골이 하얗게 뜸) | 노출 낮추기(§4), 조명 각도 조정으로 골 그림자 강화 | `prom_frac` 하향(예 0.10→0.07) |
| **검출 0개** (튜브가 있는데 빈 프레임 처리) | 튜브/배경 대비 부족(어두운 튜브 + 어두운 노출) | 노출·게인 상향, 배경판 확인 | 전역 대비 컷(20) 하향 |
| 짧은 튜브(LEN 불량)가 분리 실패 | 중앙 60% 구간에서 벗어남 | 짧은 튜브도 프레임 중앙부에 걸치도록 정렬 | 중앙 구간 비율 조정 |
| 개수 불일치가 간헐 발생 | 조명 흔들림·외광 유입 | 차광 후드 점검, 칩 청소 | — |

### 지원 요청 시 반드시 첨부할 것

- **원본(raw) 이미지** — 결과 오버레이가 아니라 `raw/` 의 원본. 대시보드 검사이력에서
  해당 배치를 열어 원본 이미지를 다운로드한다. (오버레이 이미지는 박스가 그려져 있어
  분석에 쓸 수 없다.)
- 해당 품목의 기준정보 값(개수, 외경, 촬영 레시피)과 증상 설명(과분할/미분할, 발생 빈도).

## 8. 최종 점검 체크리스트

가동 승인 전, 아래 순서대로 체크한다. **순서가 중요하다** — 앞 항목을 바꾸면 뒤 항목을
다시 해야 한다.

- [ ] **1. 기구 고정**: 카메라 브래킷·치구·배경판 볼트 고정, 위치 마킹 완료 (§2)
- [ ] **2. 조명 고정**: 확산광 설치, 차광 후드 장착, 외광 영향 없음(주/야 비교), 베드 칩 청소 (§3)
- [ ] **3. 레시피 고정**: capture_recipe 저장(고정 노출·게인·manual AF·lens_position), 워커 재시작, 화면에서 포화/흐림 없음 확인 (§4)
- [ ] **4. 캘리브레이션**: 기준 파이프로 보정 완료, 재검사 오차 확인 (§5)
- [ ] **5. 반복성**: 같은 파이프 10회 측정, 산포가 공차의 약 1/10 이내 (§5)
- [ ] **6. (다중 모드) 개수**: 기준정보 개수 = 실제 투입 개수, 배치 결과 이미지에서 튜브별 박스·번호가 정확히 N개, 개수 불일치 경고 없음 (§6)
- [ ] **7. 판정 확인**: 양품/불량 샘플 각각 투입해 OK/NG·불량코드가 기대와 일치, HMI 알람·재확인 입력 동작
- [ ] **8. 기록**: 최종 레시피 JSON·캘리브레이션 값·설치 사진을 보관(다음 재설치/이설 시 기준)

> 이후 카메라·조명·치구·해상도 중 **무엇이든 바꾸면 3번부터 다시** 진행한다.
